# ADR-0001 - Title
Author: João Pianissoli
Created At: 2025-11-22
Status: Accepted

## Context
É necessário desenvolver uma API REST para o gerenciamento de uma biblioteca digital. Para isso, é necessário decidir qual framework será utilizado na linguagem Python (pré-requisito). Dentre os requisitos técnicos necessários para esse framework estão: performance no processamento de requisições, documentação e suporte disponíveis, validação de dados robusta, proteção de rotas com a autenticação de rotas, mecanismos de documentação de rotas automático, suporte a testes automatizados e ter uma boa experiência de desenvolvimento (garantindo um código simples de dar manutenção e consistente).

## Decision
Foi escolhido o FastAPI para a implementação da API REST do sistema. Essa decisão foi motivada pelos seguintes motivos: 
1. Popularidade da ferramenta: a grande adesão indica que a ferramenta é de simples utilização, robusta e madura o suficiente para comportar os requisitos de projetos consolidados;
2. Disponibilidade de documentação: estruturas definidas pelo framework são bem documentadas e acessíveis;
3. Comunidade: é relativamente fácil encontrar soluções compartilhadas para problemas comuns de desenvolvedores que utilizam a ferramenta em seus projetos;
4. Cumpre com os requisitos técnicos desejados:
    - É performático no processamento de grandes volumes de requisições;
    - Possui natureza assíncrona implementada, o que permite que operações não-blocantes sejam realizadas "out-of-the-box", sem a necessidade da utilização de serviços de "background jobs" externos;
    - Possui mecanismos de validação de dados robustos e de simples utilização com o Pydantic;
    - Utiliza o Swagger para documentação automática de rotas;
    - Tipagem de dados extensiva;
    - Possui mecanismos de segurança para fornecer autenticação em rotas;
    - Possivelmente é o framework com a melhor experiência de desenvolvimento, proporcionando uma compreensão clara do código desenvolvido.

No processo de decisão foram também considerados as seguintes opções e seus respectivos motivos de não adesão:
- Flask: simples demais para a natureza da aplicação (não implementa o recurso assíncrono nativamente, por exemplo);
- Django: confiável e mais maduro que o FastAPI, porém, muito robusto (com muita complexidade inerente) para o projeto atual;

## Consequences
A curva de aprendizado para o framework pode representar uma consequência negativa não relevante, dado que o framework apresenta alguns padrões de implementação que devem ser seguidos e pode ser um desafio para programadores não familizados com a natureza assíncrona presente. Entretanto, muito provavelmente, esse aspecto seja mais oneroso em frameworks como Django, especialmente por conta da maior quantidade de camadas de abstração. Entretanto, a performance apresentada no processamento de requisições e a consistência do código desenvolvido, bem como todos os mecanismos de documentação automática das rotas criadas fazem do FastAPI uma escolha mais adequada para o projeto.
